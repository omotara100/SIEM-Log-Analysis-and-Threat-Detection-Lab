# SIEM Log Analysis and Threat Detection Lab

## Overview

This project demonstrates how to deploy a SIEM and perform threat detection using centralized log analysis.

## 📌 Objective
Use Splunk SIEM to detect and investigate:
- Port scanning (reconnaissance)
- Brute-force login attempts
- Suspicious PowerShell execution

## 🧱 Lab Environment
- Attacker: Kali Linux
- Target: Windows 10 VM
- SIEM: Splunk Enterprise
- Logs: Sysmon + Windows Event Logs

## Technologies Used

- Splunk SIEM  
- Sysmon  
- Windows Security Logs  
- PowerShell Logging  

## Skills Demonstrated

Log ingestion
Threat detection
Security event analysis
Detection engineering
SOC investigation workflow

## 🔍 Investigations Performed

1. Network reconnaissance (Nmap scan)
2. Failed login attempts (brute force)
3. Suspicious PowerShell execution

## Key Learning Outcomes

Centralized logging enables security analysts to detect and investigate malicious activity across systems.
